# Main Tasks of an IT Product Manager

A Product Manager's core responsibilities throughout the project lifecycle:

---

## 📋 Project Lifecycle Tasks

### 1. Inception Phase
| Task | Description |
|---|---|
| **IPM-INC-001 — Requirement Gathering** | In-depth interviews with stakeholders/customers to collect raw requirements. |
| **IPM-INC-002 — Market Research** | Analyzing competitors, industry trends, and user pain points. |
| **IPM-INC-003 — BRD Writing** | Creating the Business Requirement Document to define commercial value and goals. |
| **IPM-INC-004 — Feasibility Assessment** | Collaborating with technical teams to evaluate technical viability. |
| **IPM-INC-005 — Alignment** | Achieving consensus among business, technical, and design stakeholders. |

### 2. Requirements Phase
| Task | Description |
|---|---|
| **IPM-REQ-001 — PRD Writing** | Creating the Product Requirement Document with detailed logic, boundaries, and interactions. |
| **IPM-REQ-002 — User Story Decomposition** | Breaking down large requirements into deliverable User Stories (Definition of Ready - DoR). |
| **IPM-REQ-003 — Prioritization** | Ranking requirements using frameworks like MoSCoW or RICE. |
| **IPM-REQ-004 — Prototype Review** | Collaborating with designers to finalize UI/UX solutions. |
| **IPM-REQ-005 — Refinement Meetings** | Organizing requirement reviews to ensure shared understanding across teams. |

### 3. Sprint / Development Phase
| Task | Description |
|---|---|
| **IPM-DEV-001 — Requirement Clarification** | Responding to developer and tester queries in real-time. |
| **IPM-DEV-002 — Acceptance Criteria (AC)** | Defining the specific conditions that must be met for each feature. |
| **IPM-DEV-003 — PRD Change Management** | Managing requirement changes and assessing their impact. |
| **IPM-DEV-004 — Progress Tracking** | Monitoring delivery timelines and identifying risks early. |

### 4. QA & UAT Phase
| Task | Description |
|---|---|
| **IPM-QA-001 — UAT Organization** | Coordinating User Acceptance Testing with users/business owners. |
| **IPM-QA-002 — Bug Prioritization** | Deciding which bugs are critical and whether they block release. |
| **IPM-QA-003 — Launch Checklist** | Confirming release standards (Definition of Done - DoD). |

### 5. Release & Growth Phase
| Task | Description |
|---|---|
| **IPM-REL-001 — Release Coordination** | Planning release schedules, canary deployments, and rollback strategies. |
| **IPM-REL-002 — Data Analysis** | Analyzing KPIs after launch to verify if objectives were met. |
| **IPM-REL-003 — User Feedback** | Collecting continuous feedback for the next iteration cycle. |
| **IPM-REL-004 — Retrospectives** | Organizing project/sprint retrospectives to optimize collaboration. |
