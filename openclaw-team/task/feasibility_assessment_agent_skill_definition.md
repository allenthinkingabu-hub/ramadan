
# 帮我创建一个 会编写 可行性评估报告（Feasibility Assessment Report）的 AI Agent Skill，具体信息如下：
# 基本要求：
- 第一、请帮我收集下一个合格的 编写可行性评估报告 的 AI Agent，告诉我这个任务有什么触发机制 ，然后把这功能变成一个配置的的文件，也支持后续我们根据情况修改技能列表；
- 第二、请帮我收集下一个合格的 编写可行性评估报告 的 AI Agent ，这任务RACI矩阵， 这个RACI矩阵除了有角色名称，还有对应的任务名称，然后把这功能变成一个配置的的文件，作用一：方便 Ai Agent 每次启动知道完成这个任务的所有干系人，作用二：这个AI 完成这个任务，把这个 RACI 矩阵给 Project Manager AI Agent , 帮助 Project Manager AI Agent 触发后续任务， 也支持后续我们根据情况修改技能列表
- 第三、请帮我收集下一个合格的 编写可行性评估报告 的 AI Agent 他应该具备什么技能，然后把这功能变成一个配置的的文件，方便 Ai Agent 每次启动时候知道具备技能， 也支持后续我们根据情况修改技能列表
- 第四、请帮我收集下一个合格的 编写可行性评估报告 的 AI Agent 他应该具备什么知识体系清单，方便 Ai Agent 每次启动时候知道自己应具备的知识体系；也支持后续我们根据情况修改技能列表
- 第五、请帮我收集下一个合格的 编写可行性评估报告 的 AI Agent 应该使用什么工具，整理一份工具清单，方便 Ai Agent 每次启动时候知道自己应具备的工具；也支持后续我们根据情况修改技能列表
- 第六、请帮我收集下一个合格的 编写可行性评估报告 的 AI Agent 应该使用MCP工具，整理一份 MCP 工具清单，方便 Ai Agent 启动时候知道自己要调用什么MCP，也支持后续我们根据情况修改；
- 第七、请帮我收集下一个合格的 编写可行性评估报告 的 AI Agent 应该输出什么内容，整理一份输出内容的模版，方便 Ai Agent 启动时候知道自己要输出内容模版，严格按照要求输出，也支持后续我们根据情况修改；
- 第八、请帮我收集下一个合格的 编写可行性评估报告 的 AI Agent 应该遵循的 SOP 流程，整理一份SOP流程清单，方便 Ai Agent 启动时候知道自己要遵循的SOP流程，也支持后续我们根据情况修改；
- 第九、请帮我收集下一个合格的 编写可行性评估报告 的 AI Agent 应该满足的 质量门槛 (DoD) ，整理一份质量门槛清单，方便 Ai Agent 启动时候知道自己要满足的质量门槛，也支持后续我们根据情况修改；
- 第十、请帮我收集下一个合格的 编写可行性评估报告 的 AI Agent 应该满足的 DoR (Definition of Ready) ，满足这些条件才能启动， 这个 DoR 其中一项是需要 提供 BRD 文档，如果没有提供，请调用 brd-writer 这个 skill 去首先编写 BRD 然后再进行可行性评估，整理一份 DoR 清单，方便 Ai Agent 启动时候知道自己要满足的 DoR 才启动，也支持后续我们根据情况修改；
- 第十一、Ai Agent 要记录，每次和用户的对话记录，以问题为单位一条一条记录在文档中；
- 第十二、Ai Agent 要记录，Ai Agent 的工作记录，以时间轴为单位一条一条记录在文档中；
- 第十三、Ai Agent 根据 DoD 清单，检查这个任务是否完成，如果检查不符合要求，就返回去修改，直到符合要求；
- 第十四、**【监督 AI Agent Skill 规范】** — 为当前任务生产一个独立的 **监督 AI Agent Skill（Feasibility Assessment Supervisor Agent）**，负责对 Feasibility Assessment Writer AI Agent 的运行结果进行全程质量监控与闭环修复，具体规范如下：

  ---

  ### 14.1 角色定位
  - **Skill 名称**：`feasibility-assessment-supervisor`
  - **角色**：监督者（Quality Supervisor），独立于 Feasibility Assessment Writer Agent 之外运行，不参与可行性评估报告的编写，只负责审查与反馈。
  - **触发时机**：Feasibility Assessment Writer AI Agent 完成一轮可行性评估报告输出后，自动触发本 Skill。
  - **运行机制**：本 Skill 运行时，会自动检查 Feasibility Assessment Writer AI Agent 运行结果，并生成一份检查报告。

  ---

  ### 14.2 监督范围（检查清单）
  监督 AI Agent 将逐条检查以下 **第一至第十三步** 的执行情况，逐项判断是否已被完整执行：

  | 检查项 | 检查内容 |
  | :--- | :--- |
  | ✅ 第一 | 触发机制配置文件是否已生成 |
  | ✅ 第二 | RACI 矩阵配置文件是否已生成（含角色名称 + 对应任务名称） |
  | ✅ 第三 | 技能清单配置文件是否已生成 |
  | ✅ 第四 | 知识体系清单是否已生成 |
  | ✅ 第五 | 工具清单是否已生成 |
  | ✅ 第六 | MCP 工具清单是否已生成 |
  | ✅ 第七 | 输出内容模版是否已生成 |
  | ✅ 第八 | SOP 流程清单是否已生成 |
  | ✅ 第九 | DoD 质量门槛清单是否已生成 |
  | ✅ 第十 | DoR 清单是否已生成（含 BRD 前置依赖检查） |
  | ✅ 第十一 | 用户对话记录文档是否存在（以问题为单位逐条记录） |
  | ✅ 第十二 | AI Agent 工作记录文档是否存在（以时间轴为单位逐条记录） |
  | ✅ 第十三 | DoD 检查结果是否通过，是否已完成自动修复闭环 |

  ---

  ### 14.3 检查流程（闭环机制）

  ```
  [触发] Feasibility Assessment Writer Agent 完成输出
       ↓
  [检查] Feasibility Assessment Supervisor Agent 逐条核查第一至第十三步
       ↓
  [生成报告] 输出检查报告（见 14.4）
       ↓
  [判断] 报告通过率 = 100%？
       ├── 否 → 将报告发回 Feasibility Assessment Writer Agent，要求按报告逐项修复
       │         Feasibility Assessment Writer Agent 修复完成后，重新触发监督 Agent
       │         （重复此循环，直到 100% 通过）
       └── 是 → 调用 Project Manager AI Agent，提交完成报告
  ```

  ---

  ### 14.4 检查报告格式

  每次检查结束后，生成一份结构化报告，格式如下：

  ```markdown
  # Feasibility Assessment Supervisor 检查报告

  - 检查时间：{timestamp}
  - 检查轮次：第 {N} 轮
  - 可行性评估报告文件路径：{feasibility_assessment_file_path}

  ## 检查结果汇总

  | 检查项 | 状态 | 说明 |
  | :--- | :---: | :--- |
  | 第一：触发机制配置 | ✅ 通过 / ❌ 未通过 | {备注} |
  | 第二：RACI 矩阵配置 | ✅ 通过 / ❌ 未通过 | {备注} |
  | 第三：技能清单配置 | ✅ 通过 / ❌ 未通过 | {备注} |
  | 第四：知识体系清单 | ✅ 通过 / ❌ 未通过 | {备注} |
  | 第五：工具清单 | ✅ 通过 / ❌ 未通过 | {备注} |
  | 第六：MCP 工具清单 | ✅ 通过 / ❌ 未通过 | {备注} |
  | 第七：输出内容模版 | ✅ 通过 / ❌ 未通过 | {备注} |
  | 第八：SOP 流程清单 | ✅ 通过 / ❌ 未通过 | {备注} |
  | 第九：DoD 质量门槛清单 | ✅ 通过 / ❌ 未通过 | {备注} |
  | 第十：DoR 清单 | ✅ 通过 / ❌ 未通过 | {备注} |
  | 第十一：用户对话记录 | ✅ 通过 / ❌ 未通过 | {备注} |
  ```

  ### 14.5 完成后调用 Project Manager AI Agent
  - 当报告通过率达到 **100%** 时，监督 Agent 执行以下操作：
    1. 生成最终检查报告（标注"全部通过"）。
    2. 调用 **Project Manager AI Agent**，发送以下信息：
       - 可行性评估报告文档存放路径及文件名
       - RACI 矩阵配置（供 PM Agent 触发后续任务）
       - 最终检查报告
- 第十五、这个 Ai Agent 调用完成后，告诉 Project Manager AI Agent 完成任务，并把可行性评估报告文档存放位置及文件名称发给 Project Manager AI Agent， Project Manager AI Agent 根据 这个任务RACI矩阵去调用对应的Ai Agent 去完成 RACI 矩阵的任务。


# 功能要求：我期望一个交互性强的Ai Agent, 首先 AI Agent 拿到客户的需求或项目方案，我们称之为 Topic.

- 第一步，这个AI Agent 首先尝试理解这个任务的目的——即为什么要做可行性评估，评估的目标和范围是什么，告诉他的理解内容，然后 AI Agent 询问我，我同意后继续执行第二步，不同意要继续执行第一步，直到完全理解这个任务;

- 第二步，这个AI Agent 尝试理解这个 Topic 的业务背景、技术背景和项目约束条件，告诉他的理解内容，然后 AI Agent 询问我，我同意后继续执行第三步，不同意要继续执行第二步，直到完全理解这个 Topic;

- 第三步，根据这个 Topic 去互联网、权威知识库去调查关于这个领域的可行性评估最佳实践，告诉我行业中是怎么做可行性评估的（包括技术可行性、业务可行性、运营可行性、时间可行性、经济可行性、法律/合规可行性），并且生成问题清单，根据这个问题清单，跟我不断对话，帮助我获取全面的评估信息，形成真实的评估依据清单。

- 第四步，根据真实的评估依据清单，去互联网、权威知识库调查相关技术方案、成本数据、风险案例等信息，AI Agent 按照前面的"基本要求"完成可行性评估任务，并且根据模版输出内容。最后将其内容存放在上一级目录的 feasibility-assessment 文档中。
