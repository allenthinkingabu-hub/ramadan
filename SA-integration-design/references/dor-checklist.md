# Integration Design Agent — Definition of Ready (DoR) Checklist

## Prerequisites Before Activation

| DoR ID | Prerequisite | Description | Status |
| :--- | :--- | :--- | :---: |
| DOR-001 | Architecture Design Complete | High-level architecture design document (IA-REQ-001) is available and approved | ☐ |
| DOR-002 | Technology Stack Decided | Technology stack decision document (IA-REQ-002) is available and approved | ☐ |
| DOR-003 | Project Scope Defined | Project scope and boundaries are clearly documented | ☐ |
| DOR-004 | Stakeholders Identified | All relevant stakeholders are identified per RACI matrix | ☐ |
| DOR-005 | Existing System Inventory | Current system landscape is documented (if applicable) | ☐ |
| DOR-006 | Security Requirements Available | Security requirements and policies are accessible | ☐ |
| DOR-007 | API Standards Defined | Organization API standards/guidelines are available (if any) | ☐ |
| DOR-008 | Business Requirements Clear | Business requirements driving integration needs are documented | ☐ |
| DOR-009 | Access to Resources | Agent has access to required tools, MCP servers, and file system | ☐ |
| DOR-010 | User Available | A user/stakeholder is available for interactive Q&A sessions | ☐ |

## Verification Process

1. Before starting the Integration Design task, verify all DoR items
2. Mark each item as ✅ (ready) or ❌ (not ready)
3. If any mandatory item (DOR-001 through DOR-004) is not ready, do not proceed
4. If optional items (DOR-005 through DOR-007) are not ready, proceed with noted gaps
5. DOR-008 through DOR-010 are required for effective execution

## Configuration Notes

- Add new prerequisites by appending rows.
- Items DOR-001 to DOR-004 are mandatory; others are recommended.
