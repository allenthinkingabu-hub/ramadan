    # Main Tasks of a Data Analyst

A Data Analyst's core responsibilities throughout the project lifecycle:

---

## 📋 Project Lifecycle Tasks

### 1. Inception Phase
| Task | Description |
|---|---|
| **#📋PLT-INC-001 — Business Context Understanding** | Meeting with stakeholders to understand business objectives, key questions, and the decisions data needs to support. |
| **#📋PLT-INC-002 — Data Source Identification** | Inventorying available data sources, databases, APIs, spreadsheets, and third-party data relevant to the project. |
| **#📋PLT-INC-003 — Data Quality Assessment** | Profiling existing data for completeness, accuracy, consistency, and timeliness to gauge analytical feasibility. |
| **#📋PLT-INC-004 — KPI & Metrics Definition** | Collaborating with PM and business stakeholders to define measurable KPIs, success metrics, and target benchmarks. |
| **#📋PLT-INC-005 — Analytical Approach Scoping** | Proposing the analytical methods, tools, and level of effort required to deliver actionable insights. |

### 2. Requirements Phase
| Task | Description |
|---|---|
| **#📋PLT-REQ-001 — Reporting Requirements Gathering** | Documenting detailed reporting needs including dimensions, measures, filters, drill-downs, and refresh frequency. |
| **#📋PLT-REQ-002 — Dashboard & Visualization Design** | Wireframing dashboards and report layouts in collaboration with stakeholders and UI/UX designers. |
| **#📋PLT-REQ-003 — Data Dictionary Development** | Creating a data dictionary defining field names, business definitions, data types, sources, and transformation rules. |
| **#📋PLT-REQ-004 — Data Access & Security Requirements** | Defining role-based access controls, data sensitivity classifications, and row/column-level security for reports. |
| **#📋PLT-REQ-005 — Acceptance Criteria for Analytics** | Establishing clear acceptance criteria for reports, dashboards, and analytical deliverables with stakeholder agreement. |

### 3. Sprint / Development Phase
| Task | Description |
|---|---|
| **#📋PLT-DEV-001 — Data Extraction & Preparation** | Writing SQL queries and scripts to extract, clean, transform, and prepare data sets for analysis. |
| **#📋PLT-DEV-002 — Exploratory Data Analysis (EDA)** | Performing statistical analysis, trend identification, correlation analysis, and anomaly detection on prepared data. |
| **#📋PLT-DEV-003 — Dashboard & Report Development** | Building interactive dashboards and reports using BI tools (Tableau, Power BI, Looker, Superset). |
| **#📋PLT-DEV-004 — Ad-Hoc Analysis** | Responding to stakeholder requests for on-demand deep-dive analyses, data pulls, and quick-turn insights. |
| **#📋PLT-DEV-005 — Data Validation & Reconciliation** | Cross-referencing analytical outputs against source systems and known benchmarks to ensure accuracy. |

### 4. QA & UAT Phase
| Task | Description |
|---|---|
| **#📋PLT-QA-001 — Report & Dashboard Testing** | Validating dashboard calculations, filters, drill-downs, and visual accuracy against expected results. |
| **#📋PLT-QA-002 — Data Accuracy Verification** | Reconciling report outputs with source data, financial records, or operational systems for correctness. |
| **#📋PLT-QA-003 — User Acceptance Walkthrough** | Conducting guided walkthroughs with business users to validate reports meet their needs and expectations. |
| **#📋PLT-QA-004 — Performance Testing** | Verifying dashboard load times, query performance, and responsiveness with production-scale data volumes. |
| **#📋PLT-QA-005 — Documentation & Training Materials** | Preparing user guides, data glossaries, and training content for end-user self-service adoption. |

### 5. Release & Growth Phase
| Task | Description |
|---|---|
| **#📋PLT-REL-001 — Report Deployment & Distribution** | Publishing dashboards to production workspaces, configuring scheduled refreshes, and setting up email subscriptions. |
| **#📋PLT-REL-002 — Post-Launch Monitoring** | Monitoring report usage, adoption rates, query performance, and data refresh reliability after go-live. |
| **#📋PLT-REL-003 — Insight Generation & Storytelling** | Translating data findings into actionable business narratives, presentations, and executive summaries. |
| **#📋PLT-REL-004 — Continuous Analysis & Iteration** | Refining reports, adding new metrics, and evolving dashboards based on user feedback and changing business needs. |
| **#📋PLT-REL-005 — Self-Service Enablement** | Training business users on self-service analytics, building reusable data sets, and maintaining semantic models. |
