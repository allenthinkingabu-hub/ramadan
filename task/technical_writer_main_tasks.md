# Main Tasks of a Technical Writer

A Technical Writer's core responsibilities throughout the project lifecycle:

---

## 📋 Project Lifecycle Tasks

### 1. Inception Phase
| Task | Description |
|---|---|
| **Documentation Needs Assessment** | Identifying the types, scope, and audience of documentation required for the project (end-user, developer, operations). |
| **Audience Analysis** | Profiling target readers to determine technical depth, language level, format preferences, and accessibility needs. |
| **Documentation Standards & Style Guide** | Establishing or adopting writing standards, style guides (Microsoft, Google, custom), terminology conventions, and templates. |
| **Toolchain Selection** | Selecting documentation platforms and tools (Confluence, GitBook, ReadTheDocs, MkDocs, Docusaurus, Madcap Flare). |
| **Documentation Plan & Timeline** | Creating a documentation plan with deliverables, milestones, review cycles, and alignment to project delivery schedule. |

---

### 2. Requirements Phase
| Task | Description |
|---|---|
| **Information Architecture** | Designing the documentation structure including navigation hierarchy, content grouping, and cross-referencing strategy. |
| **Content Outline Development** | Producing detailed outlines for each deliverable (user guides, API docs, admin manuals, release notes, FAQs). |
| **Subject Matter Expert (SME) Coordination** | Scheduling interviews and review sessions with developers, architects, and PMs to gather accurate technical content. |
| **Terminology & Glossary Creation** | Building a project glossary defining key terms, acronyms, and domain-specific language for consistency across all documents. |
| **Template & Component Design** | Creating reusable document templates, admonition blocks, code sample formats, and diagram standards. |

---

### 3. Sprint / Development Phase
| Task | Description |
|---|---|
| **User Documentation Drafting** | Writing end-user guides, getting started tutorials, how-to articles, and feature walkthroughs aligned with sprint deliverables. |
| **API & Developer Documentation** | Documenting API references (OpenAPI/Swagger), SDK guides, code samples, authentication flows, and integration tutorials. |
| **Admin & Operations Documentation** | Creating installation guides, configuration references, troubleshooting runbooks, and system administration manuals. |
| **Docs-as-Code Workflow** | Managing documentation in version control (Git), writing in Markdown/AsciiDoc, and integrating doc builds into CI/CD pipelines. |
| **Review & Feedback Cycles** | Conducting peer reviews, SME technical reviews, and editorial reviews to ensure accuracy, clarity, and completeness. |

---

### 4. QA & UAT Phase
| Task | Description |
|---|---|
| **Documentation Accuracy Validation** | Testing documented procedures, screenshots, and instructions against the actual product to verify correctness. |
| **Usability Testing** | Conducting documentation usability tests with representative users to identify navigation issues and comprehension gaps. |
| **Localization & Accessibility Review** | Reviewing content for localization readiness (i18n) and accessibility compliance (alt text, screen reader compatibility, plain language). |
| **Release Notes Preparation** | Compiling release notes summarizing new features, enhancements, bug fixes, known issues, and breaking changes. |
| **Final Editorial Review** | Performing final proofreading, style guide compliance checks, and link validation before publication. |

---

### 5. Release & Growth Phase
| Task | Description |
|---|---|
| **Documentation Publishing & Deployment** | Publishing documentation to production portals, help centers, developer hubs, and knowledge bases. |
| **User Feedback & Analytics** | Monitoring documentation usage analytics (page views, search queries, bounce rates) and collecting reader feedback. |
| **Content Maintenance & Updates** | Updating documentation to reflect product changes, patches, deprecations, and new feature releases. |
| **Knowledge Base Expansion** | Building FAQ sections, troubleshooting articles, and community-contributed content based on support tickets and user questions. |
| **Documentation Retrospective** | Reviewing documentation process effectiveness, identifying improvement areas, and refining workflows for future projects. |

---

## 🎯 Core Deliverables

```
Documentation Plan → Style Guide & Glossary → Information Architecture → User Guides & Tutorials → API & Developer Docs → Admin & Operations Manuals → Release Notes → Localization-Ready Content → Published Documentation Portal → Usage Analytics Reports → Documentation Maintenance Log
```
