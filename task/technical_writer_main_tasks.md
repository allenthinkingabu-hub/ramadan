# Main Tasks of a Technical Writer

A Technical Writer's core responsibilities throughout the project lifecycle:

---

## 📋 Project Lifecycle Tasks

### 1. Inception Phase
| Task | Description |
|---|---|
| **TW-INC-001 — Documentation Needs Assessment** | Identifying the types, scope, and audience of documentation required for the project (end-user, developer, operations). |
| **TW-INC-002 — Audience Analysis** | Profiling target readers to determine technical depth, language level, format preferences, and accessibility needs. |
| **TW-INC-003 — Documentation Standards & Style Guide** | Establishing or adopting writing standards, style guides (Microsoft, Google, custom), terminology conventions, and templates. |
| **TW-INC-004 — Toolchain Selection** | Selecting documentation platforms and tools (Confluence, GitBook, ReadTheDocs, MkDocs, Docusaurus, Madcap Flare). |
| **TW-INC-005 — Documentation Plan & Timeline** | Creating a documentation plan with deliverables, milestones, review cycles, and alignment to project delivery schedule. |

### 2. Requirements Phase
| Task | Description |
|---|---|
| **TW-REQ-001 — Information Architecture** | Designing the documentation structure including navigation hierarchy, content grouping, and cross-referencing strategy. |
| **TW-REQ-002 — Content Outline Development** | Producing detailed outlines for each deliverable (user guides, API docs, admin manuals, release notes, FAQs). |
| **TW-REQ-003 — Subject Matter Expert (SME) Coordination** | Scheduling interviews and review sessions with developers, architects, and PMs to gather accurate technical content. |
| **TW-REQ-004 — Terminology & Glossary Creation** | Building a project glossary defining key terms, acronyms, and domain-specific language for consistency across all documents. |
| **TW-REQ-005 — Template & Component Design** | Creating reusable document templates, admonition blocks, code sample formats, and diagram standards. |

### 3. Sprint / Development Phase
| Task | Description |
|---|---|
| **TW-DEV-001 — User Documentation Drafting** | Writing end-user guides, getting started tutorials, how-to articles, and feature walkthroughs aligned with sprint deliverables. |
| **TW-DEV-002 — API & Developer Documentation** | Documenting API references (OpenAPI/Swagger), SDK guides, code samples, authentication flows, and integration tutorials. |
| **TW-DEV-003 — Admin & Operations Documentation** | Creating installation guides, configuration references, troubleshooting runbooks, and system administration manuals. |
| **TW-DEV-004 — Docs-as-Code Workflow** | Managing documentation in version control (Git), writing in Markdown/AsciiDoc, and integrating doc builds into CI/CD pipelines. |
| **TW-DEV-005 — Review & Feedback Cycles** | Conducting peer reviews, SME technical reviews, and editorial reviews to ensure accuracy, clarity, and completeness. |

### 4. QA & UAT Phase
| Task | Description |
|---|---|
| **TW-QA-001 — Documentation Accuracy Validation** | Testing documented procedures, screenshots, and instructions against the actual product to verify correctness. |
| **TW-QA-002 — Usability Testing** | Conducting documentation usability tests with representative users to identify navigation issues and comprehension gaps. |
| **TW-QA-003 — Localization & Accessibility Review** | Reviewing content for localization readiness (i18n) and accessibility compliance (alt text, screen reader compatibility, plain language). |
| **TW-QA-004 — Release Notes Preparation** | Compiling release notes summarizing new features, enhancements, bug fixes, known issues, and breaking changes. |
| **TW-QA-005 — Final Editorial Review** | Performing final proofreading, style guide compliance checks, and link validation before publication. |

### 5. Release & Growth Phase
| Task | Description |
|---|---|
| **TW-REL-001 — Documentation Publishing & Deployment** | Publishing documentation to production portals, help centers, developer hubs, and knowledge bases. |
| **TW-REL-002 — User Feedback & Analytics** | Monitoring documentation usage analytics (page views, search queries, bounce rates) and collecting reader feedback. |
| **TW-REL-003 — Content Maintenance & Updates** | Updating documentation to reflect product changes, patches, deprecations, and new feature releases. |
| **TW-REL-004 — Knowledge Base Expansion** | Building FAQ sections, troubleshooting articles, and community-contributed content based on support tickets and user questions. |
| **TW-REL-005 — Documentation Retrospective** | Reviewing documentation process effectiveness, identifying improvement areas, and refining workflows for future projects. |
