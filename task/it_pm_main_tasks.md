# Main Tasks of an IT Product Manager

A Product Manager's core responsibilities throughout the project lifecycle:

---

## 📋 Project Lifecycle Tasks

### 1. Inception Phase
| Task | Description |
|---|---|
| **Requirement Gathering** | In-depth interviews with stakeholders/customers to collect raw requirements. |
| **Market Research** | Analyzing competitors, industry trends, and user pain points. |
| **BRD Writing** | Creating the Business Requirement Document to define commercial value and goals. |
| **Feasibility Assessment** | Collaborating with technical teams to evaluate technical viability. |
| **Alignment** | Achieving consensus among business, technical, and design stakeholders. |

---

### 2. Requirements Phase
| Task | Description |
|---|---|
| **PRD Writing** | Creating the Product Requirement Document with detailed logic, boundaries, and interactions. |
| **User Story Decomposition** | Breaking down large requirements into deliverable User Stories (Definition of Ready - DoR). |
| **Prioritization** | Ranking requirements using frameworks like MoSCoW or RICE. |
| **Prototype Review** | Collaborating with designers to finalize UI/UX solutions. |
| **Refinement Meetings** | Organizing requirement reviews to ensure shared understanding across teams. |

---

### 3. Sprint / Development Phase
| Task | Description |
|---|---|
| **Requirement Clarification** | Responding to developer and tester queries in real-time. |
| **Acceptance Criteria (AC)** | Defining the specific conditions that must be met for each feature. |
| **PRD Change Management** | Managing requirement changes and assessing their impact. |
| **Progress Tracking** | Monitoring delivery timelines and identifying risks early. |

---

### 4. QA & UAT Phase
| Task | Description |
|---|---|
| **UAT Organization** | Coordinating User Acceptance Testing with users/business owners. |
| **Bug Prioritization** | Deciding which bugs are critical and whether they block release. |
| **Launch Checklist** | Confirming release standards (Definition of Done - DoD). |

---

### 5. Release & Growth Phase
| Task | Description |
|---|---|
| **Release Coordination** | Planning release schedules, canary deployments, and rollback strategies. |
| **Data Analysis** | Analyzing KPIs after launch to verify if objectives were met. |
| **User Feedback** | Collecting continuous feedback for the next iteration cycle. |
| **Retrospectives** | Organizing project/sprint retrospectives to optimize collaboration. |

---

## 🎯 Core Deliverables

```
BRD → PRD → User Stories → Prototype Approval → Requirement Review → UAT Report → Release Report
```
