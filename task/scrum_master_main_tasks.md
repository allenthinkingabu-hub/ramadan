# Main Tasks of a Scrum Master

A Scrum Master's core responsibilities throughout the project lifecycle:

---

## 📋 Project Lifecycle Tasks

### 1. Inception Phase
| Task | Description |
|---|---|
| **SM-INC-001 — Agile Framework Setup** | Selecting and tailoring the Agile methodology (Scrum, Kanban, SAFe) to fit the project context and team size. |
| **SM-INC-002 — Team Formation** | Facilitating team onboarding, defining roles, and establishing working agreements and team norms. |
| **SM-INC-003 — Tooling & Workspace Setup** | Configuring project management tools (Jira, Azure DevOps, Trello) with boards, workflows, and backlogs. |
| **SM-INC-004 — Stakeholder Alignment** | Educating stakeholders on Agile principles, setting expectations for ceremonies, cadence, and transparency. |
| **SM-INC-005 — Definition of Ready & Done** | Collaborating with the team to establish clear Definition of Ready (DoR) and Definition of Done (DoD) criteria. |

### 2. Requirements Phase
| Task | Description |
|---|---|
| **SM-REQ-001 — Backlog Refinement Facilitation** | Organizing and facilitating backlog grooming sessions to ensure stories are well-defined and estimated. |
| **SM-REQ-002 — Sprint Planning** | Leading sprint planning ceremonies to select sprint goals, commit to backlog items, and allocate capacity. |
| **SM-REQ-003 — Estimation Facilitation** | Guiding the team through estimation techniques (Planning Poker, T-shirt sizing, affinity mapping). |
| **SM-REQ-004 — Dependency Identification** | Mapping cross-team dependencies and coordinating with other Scrum Masters or program-level roles. |
| **SM-REQ-005 — Capacity Planning** | Tracking team availability, velocity trends, and leave schedules to set realistic sprint commitments. |

### 3. Sprint / Development Phase
| Task | Description |
|---|---|
| **SM-DEV-001 — Daily Standup Facilitation** | Running focused daily stand-ups to surface blockers, sync progress, and maintain team alignment. |
| **SM-DEV-002 — Impediment Removal** | Actively identifying and resolving blockers by escalating issues, coordinating resources, or negotiating priorities. |
| **SM-DEV-003 — Sprint Progress Tracking** | Monitoring burndown/burnup charts, sprint health metrics, and flagging risks to sprint goal delivery. |
| **SM-DEV-004 — Team Coaching** | Coaching team members on Agile practices, self-organization, collaboration, and continuous improvement. |
| **SM-DEV-005 — Scope Change Protection** | Shielding the team from mid-sprint scope creep and facilitating proper change management through the Product Owner. |

### 4. QA & UAT Phase
| Task | Description |
|---|---|
| **SM-QA-001 — UAT Coordination** | Facilitating UAT scheduling, stakeholder availability, and feedback collection within sprint timelines. |
| **SM-QA-002 — Bug Triage Facilitation** | Organizing bug triage sessions with PM, QA, and developers to prioritize defect resolution. |
| **SM-QA-003 — Release Readiness Check** | Verifying all DoD criteria are met, acceptance tests pass, and release blockers are resolved. |
| **SM-QA-004 — Cross-Team Sync** | Coordinating with other teams on integration testing schedules, shared environment access, and release windows. |

### 5. Release & Growth Phase
| Task | Description |
|---|---|
| **SM-REL-001 — Sprint Review Facilitation** | Organizing sprint demos for stakeholders to showcase completed work and gather feedback. |
| **SM-REL-002 — Sprint Retrospective** | Facilitating retrospectives to identify what went well, what needs improvement, and actionable commitments. |
| **SM-REL-003 — Metrics & Reporting** | Compiling and presenting Agile metrics (velocity, cycle time, lead time, sprint predictability) to stakeholders. |
| **SM-REL-004 — Process Improvement** | Implementing retrospective action items and continuously refining team workflows and ceremonies. |
| **SM-REL-005 — Knowledge Transfer** | Documenting team practices, playbooks, and lessons learned for organizational Agile maturity growth. |
