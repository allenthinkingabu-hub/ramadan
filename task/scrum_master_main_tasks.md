# Main Tasks of a Scrum Master

A Scrum Master's core responsibilities throughout the project lifecycle:

---

## 📋 Project Lifecycle Tasks

### 1. Inception Phase
| Task | Description |
|---|---|
| **Agile Framework Setup** | Selecting and tailoring the Agile methodology (Scrum, Kanban, SAFe) to fit the project context and team size. |
| **Team Formation** | Facilitating team onboarding, defining roles, and establishing working agreements and team norms. |
| **Tooling & Workspace Setup** | Configuring project management tools (Jira, Azure DevOps, Trello) with boards, workflows, and backlogs. |
| **Stakeholder Alignment** | Educating stakeholders on Agile principles, setting expectations for ceremonies, cadence, and transparency. |
| **Definition of Ready & Done** | Collaborating with the team to establish clear Definition of Ready (DoR) and Definition of Done (DoD) criteria. |

---

### 2. Requirements Phase
| Task | Description |
|---|---|
| **Backlog Refinement Facilitation** | Organizing and facilitating backlog grooming sessions to ensure stories are well-defined and estimated. |
| **Sprint Planning** | Leading sprint planning ceremonies to select sprint goals, commit to backlog items, and allocate capacity. |
| **Estimation Facilitation** | Guiding the team through estimation techniques (Planning Poker, T-shirt sizing, affinity mapping). |
| **Dependency Identification** | Mapping cross-team dependencies and coordinating with other Scrum Masters or program-level roles. |
| **Capacity Planning** | Tracking team availability, velocity trends, and leave schedules to set realistic sprint commitments. |

---

### 3. Sprint / Development Phase
| Task | Description |
|---|---|
| **Daily Standup Facilitation** | Running focused daily stand-ups to surface blockers, sync progress, and maintain team alignment. |
| **Impediment Removal** | Actively identifying and resolving blockers by escalating issues, coordinating resources, or negotiating priorities. |
| **Sprint Progress Tracking** | Monitoring burndown/burnup charts, sprint health metrics, and flagging risks to sprint goal delivery. |
| **Team Coaching** | Coaching team members on Agile practices, self-organization, collaboration, and continuous improvement. |
| **Scope Change Protection** | Shielding the team from mid-sprint scope creep and facilitating proper change management through the Product Owner. |

---

### 4. QA & UAT Phase
| Task | Description |
|---|---|
| **UAT Coordination** | Facilitating UAT scheduling, stakeholder availability, and feedback collection within sprint timelines. |
| **Bug Triage Facilitation** | Organizing bug triage sessions with PM, QA, and developers to prioritize defect resolution. |
| **Release Readiness Check** | Verifying all DoD criteria are met, acceptance tests pass, and release blockers are resolved. |
| **Cross-Team Sync** | Coordinating with other teams on integration testing schedules, shared environment access, and release windows. |

---

### 5. Release & Growth Phase
| Task | Description |
|---|---|
| **Sprint Review Facilitation** | Organizing sprint demos for stakeholders to showcase completed work and gather feedback. |
| **Sprint Retrospective** | Facilitating retrospectives to identify what went well, what needs improvement, and actionable commitments. |
| **Metrics & Reporting** | Compiling and presenting Agile metrics (velocity, cycle time, lead time, sprint predictability) to stakeholders. |
| **Process Improvement** | Implementing retrospective action items and continuously refining team workflows and ceremonies. |
| **Knowledge Transfer** | Documenting team practices, playbooks, and lessons learned for organizational Agile maturity growth. |

---

## 🎯 Core Deliverables

```
Agile Framework & Working Agreements → Sprint Plans → Burndown/Burnup Charts → Impediment Logs → Sprint Review Notes → Retrospective Action Items → Agile Metrics Reports → Process Improvement Backlog
```
